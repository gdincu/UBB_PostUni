Ce este XML?

  • Este un limbaj de markup (markup language):
    • destinat stocării și transportului de date;
    • folosit pentru structurarea și auto-descrierea datelor;
  • Descrie sintaxa și semantica datelor.
  • Este un meta-limbaj, folosit în crearea altor limbaje de marcare (XHTML, XSLT, RDF, SMIL, etc.);
  • Este ușor de citit pentru oameni și mașini;
  • Este independent de software și hardware, deoarece este text simplu (plain text).

Prologul XML

  • Declarația XML este opțională:

Elementele XML

  • este permis un singur element rădăcină;
  • Etichetă/Tag de start: permite specificarea atributelor;
  • Etichetă/Tag de sfârșit: trebuie să se potrivească cu numele, (case)-ul și nivelul de încadrare (nesting level) al etichetei de început;
  • numele trebuie să înceapă cu o literă sau cu caracterul de subliniere (underline) și trebuie să conțină numai litere, numere, cratime, puncte și caracterul de subliniere (underline).

  • XML Schema Definition (XSD) is the successor of DTDs
  • like a DTD, an XSD defines:
    • the elements which appear in the XML doc and their attributes
    • the order/hierarchical structure of these elements
    • the number of child elements of a specific type
    • whether the element is empty or it has content
    • default and fixed values for elements and attributes
  • additional to DTDs, XSDs:
    • support basic data types (e.g. numerical, date, string etc.)
    • support namespaces (for solving collisions)
    • use XML syntax

Vizualizarea XML

  • dacă un document XML are erori (adică nu este bine format), acesta nu va fi afișat în browser, spre deosebire de HTML, care va fi afișat dacă are erori (standardul XML W3C specifică faptul că un parser XML ar trebui să se oprească la găsirea unei erori);
  • în mod implicit un browsers afișează stuctura arborescentă a unui fișier XML, deoarece XML nu conține informații de afișare/formatare ;
  • un XML poate fi afișat diferit (formatat) cu XSLT și/sau CSS.
