HTTP (HyperText Transfer Protocol)

  • este un protocol cerere-răspuns (Request-Response)
  • nu are stare (nu menține starea unei sesiuni) și asincron (un document HTML este încărcat asincron de browser în momentul în care sunt disponibile părți din acesta)
  • este standardizată de IETF (RFC 2616)
The Hypertext Transfer Protocol (HTTP) is an application-level protocol for distributed, collaborative, hypermedia information systems. This is the foundation for data communication for the World Wide Web (i.e. internet) since 1990. HTTP is a generic and stateless protocol which can be used for other purposes as well using extensions of its request methods, error codes, and headers.

HTTP - Versiuni

HTTP/0.9

HTTP/1.0
o nouă conexiune este utilizată pentru fiecare schimb de cereri/răspunsuri (versiunea creată în 1996 și standardizată în RFC 1945).
HTTP/1.1
O conexiune poate fi utilizată pentru una sau mai multe schimburi de cereri/răspunsuri.

HTTP-Elemente de bază

A browser is an HTTP client because it sends requests to an HTTP server (Web server), which then sends responses back to the client.

The communication usually takes place over TCP/IP, but any reliable transport can be used. The default port for TCP/IP is 80, but other ports can also be used.

Communication between a host and a client occurs, via a request/response pair. The client initiates an HTTP request message, which is serviced through a HTTP response message in return.

Ce sunt "resursele"?

HTTP este folosit pentru a transfera resurse:
  • un fișier;
  • un rezultat de interogare generat dinamic;
  • ieșirea (output-ul) unui script CGI;
  • un document disponibil în mai multe limbi;
  • altceva.

O resursă este o informație care poate fi identificată printr-o adresă URL (este R-ul din URL).

2 HTTP-coduri de stare Status-Codes

  • Successful - Informează clientul că cererea a fost procesată cu succes. 200 OK este cea mai des întâlnită.
  • Redirection - solicită clientului să ia măsuri suplimentare. Cea mai obișnuită situație de utilizare este să treceți la o altă adresă URL pentru a obține resursa.
  • Client Error - serverul consideră că este o eroare la client, fie prin solicitarea unei resurse nevalide, fie prin crearea unei solicitări incorecte. Cel mai comun cod din această clasă este 404 Not Found.
  • Server Error - folosit pentru a indica o eroare de server în timpul procesării cererii. Cel mai frecvent utilizat cod de eroare este 500 Internal Server Error.

2 Răspunsul HTTP header

Accept-Ranges indică unitatea care poate fi utilizată pentru a defini un interval (range).
Content-Length indică dimensiunea corpului răspunsului, în număr de octeți,
trimisă destinatarului.
Content-Type Indică tipul media al resurselor.
Content-Security-Policy controlează resursele pe care agentul utilizator le poate încărca
pentru o anumită pagină.
Date Conține data și ora la care s-a generat răspunsul.
ETag este un validator; un șir unic care identifică versiunea resursei.
Last-Modified este un validator; ultima dată de modificare a resursei
Strict-Transport-Security specifică faptul că un site trebuie accesat numai folosind HTTPS
